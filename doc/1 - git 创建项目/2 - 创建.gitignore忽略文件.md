
## 作用
- `.gitignore` 用来告诉 Git：哪些文件/目录“永远不要被版本管理”

> 典型不该提交的东西包括：
> - 本地环境生成的文件（缓存、日志）
> - IDE / 编辑器配置
> - 虚拟环境、依赖目录
> - 操作系统自动生成的文件


## 生效前提
- `.gitignore` 只对**尚未被 Git 追踪的文件**生效；
- 如果某个文件已经被 `git add` / `git commit` 过，则`.gitignore` 对它是无效的；


## 创建文件
- 在项目根目录创建文件，文件名必须是：`.gitignore`（没有任何后缀）

```
my-project/
├── .gitignore
├── main.py
├── README.md
└── ...
```

- 打开该文件，即可编辑；


## 配置文件
### VS Code / IDE 文件

```bash
# VS Code

.idea/
```

```bash
# Python 缓存
__pycache__/

```

```bash
# 本地配置

# 数据 / 模型 / 临时文件
data/

```


## 语法
- 忽略文件：`test.log`
- 忽略某一类文件：`*.log`
- 忽略目录（推荐写法）：`venv/`
- 忽略所有某类文件，但保留一个：`*.txt !important.txt`
- 注释（给未来的你看的）：`# 这是缓存文件 __pycache__/`


## 提交 `.gitignore`
### 查看修改状态
输入：`git status`
会发现被忽略的文件不再出现在 Untracked files 中。

### 提交

```bash
git add .gitignore
git commit -m "添加.gitignore，忽略本地环境与缓存文件" 
git push
```


## 移除不该提交的文件
### 从Git中移除

假设 `__pycache__/` 为误提交文件

```bash
git rm -r --cached __pycache__/
```

### 提交修改

```
git add .gitignore 
git commit -m "移除缓存文件并加入.gitignore" 
git push
```


## 日常用法
- 查看修改状态：`git status`
- 看到不该出现的文件时，将其加入 `.gitignore` 中；
- `git add .gitignore ` -> `git commit -m "xxx"` -> `git push`

一般为手动决定文件是否需提交。
