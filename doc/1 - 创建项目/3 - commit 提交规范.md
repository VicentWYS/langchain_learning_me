
## 提交格式

```
<type>(<scope>): <subject>
```


## 改动类型

```
type:
  feat     新功能
  fix      Bug 修复
  docs     文档变更
  style    代码格式（不影响逻辑）
  refactor 重构（非功能性改动）
  perf     性能优化
  test     测试相关
  build    构建系统或依赖变更
  ci       CI 配置变更
  chore    其他杂项
  revert   回滚提交
```

## 格式
- 使用祈使句（add / fix / update）
- 不以大写字母开头
- 结尾不加句号
- 不超过 50 个字符
